## Affected Pages/Routes
- `GET /api/signals/recent?days=N` — 新增时间线 API，仪表盘信号页切换为跨季度时间线

## Key Interactions to Verify
- 每日 job 正常运行（17:00 触发 → 多源抓取 → 增量入库 → 评分 → 无 LLM 调用）
- 季度 job 不受影响（13F 抓取 → Kimi 分析 → 报告生成照常）
- 合规过滤器：带归因转述（"高盛目标价"）通过，无归因建议（"建议买入"）被拦截
- 单源失败时每日 job 继续运行，errors 字段记录失败源，前端提示"部分源不可用"

## Edge Cases
- 每日 job 首次运行（watermark=NULL，全量抓取）
- signal_fingerprint 冲突时仅更新评分字段，内容字段保持不变
- 90 天保留策略：边界日期（刚好 90 天）的信号应保留
- macro_view 空 companies 列表——scorer 不应因空列表报错
- 源无日期字段时，fingerprint 以首次见到日为准并固定
- 每日 job 与季度 job 写操作不重叠（13F 信号只由季度 job 写入）

- 存储语义改造（init_db 迁移 → upsert 逻辑 → 时间线 API → 90 天清理）

- 13D/G 源（CIK 路径 + EFTS 路径各自 watermark → max_items 截断 → SEC 异常不崩溃）
- 合规护栏（check_content 测试 → filter_for_report 测试 → [→EVAL] LLM 提示词约束验证）
- 评分器扩展（新权重 13D/G=1.1, research_view=1.0, macro_view=0.7）
- 回归验证：全部 149+ 现有测试通过，季度 job 行为不变
