### 2.4 `src/main.py`

`run_pipeline()` 新增逻辑：

1. 生成报告后，从 `src.storage` 读取上一季度持仓。
2. 调用 `QuarterComparator` 得到变化统计。
3. 构造 `Notification`。
4. 调用 `Notifier.send()`，**成功后**调用 `mark_notification_sent(quarter)`。
5. 任何异常只记录日志，不阻断主流程。

## 3. 错误路径与处理

| 错误场景 | 触发位置 | 处理方式 | 用户可见 | 测试 |
|---|---|---|---|---|
| `FEISHU_WEBHOOK` 未配置 | `Notifier.send()` | 静默跳过，函数直接返回 | 无 | ✅ test_skip_when_not_configured |
| 飞书 5xx | `_send_feishu` | 指数退避重试，最多 3 次请求 | 无 | ✅ test_retry_on_5xx |
| 飞书 429 | `_send_feishu` | 指数退避重试 | 无 | ✅ test_retry_on_429 |
| 飞书 4xx（非 429） | `_send_feishu` | 不重试，抛出异常 | 无 | ✅ test_no_retry_on_4xx |
| 飞书 200 但 `code != 0` | `_send_feishu` | 抛出异常，由外层重试逻辑处理 | 无 | ✅ test_feishu_app_error |
| 网络超时/连接错误 | `httpx` | 重试 3 次 | 无 | ✅ test_retry_on_timeout |
| 全部重试失败 | `main.py` | 记录 ERROR 日志，**不写入** `sent_notifications` | 无 | ✅ test_logs_after_exhausted |
| 同一 quarter 重复触发 | `mark_notification_sent` | 返回 False，跳过发送 | 无 | ✅ test_dedup_repeat_quarter |
| `PUBLIC_BASE_URL` 未配置 | `main.py` | 通知中省略链接，记录 WARNING | 消息无链接 | ✅ test_missing_base_url_warning |
| 首次运行无上一季度 | `main.py` | `summary=None`，只显示总市值 | 消息无变化统计 | ✅ test_first_run_summary |
| 通知发送异常 | `main.py` | `try/except Exception`，记录日志，不阻断 | 无 | ✅ test_notification_failure_non_blocking |
| 消息超长 | `_send_feishu` | 按字节截断至飞书限制以内，并追加省略号 | 无 | ✅ test_truncate_long_message |

## 6. NOT in scope

- 邮件、钉钉、Telegram 通知实现：现有 `Notifier` 类中仍是占位，本次不改。
- 飞书交互卡片/富文本消息：第一版使用纯文本 `msg_type="text"`。
- Webhook 签名验证：企业飞书自定义签名需求后续扩展。
- 用户级通知偏好/按渠道路由：第一版只按环境变量开关。
- 13F 修正文件（13F-HR/A）的再次通知：修正后用户通过 Web 查看最新报告。
- 通知失败后的持久化重试队列：本次仅记录日志，失败会在下一次 pipeline 运行（下周）时自然重试。

## 8. 设计决策

- 不重写现有 `Notifier` 抽象，只在 `_send_feishu` 里实现最小功能，符合 YAGNI。
- 重试逻辑内聚在 `notifier.py`，不抽取通用装饰器（当前只有 3 处重试：SEC、Anthropic、通知，抽象收益小）。
- 去重标记**在 HTTP 发送成功后**写入，避免失败季度被永久跳过。
- 去重仍使用 SQLite 原子 `INSERT`，避免并发重复推送。
- 报告链接依赖 `PUBLIC_BASE_URL`；未配置时降级为无链接消息，不阻塞发送。
- 消息体按字节截断到 15KB（飞书文本消息安全上限约 16KB），超长时追加省略号。

## 9. Outside Voice Findings

独立评审（Claude subagent）发现 4 个问题，已在计划中处理：

1. **去重标记顺序**：原设计在发送前标记，会导致发送失败后永久沉默。已改为**发送成功后写入** `sent_notifications`。
2. **失败无持久化重试队列**：已明确列入 `NOT in scope`，并说明下一次 pipeline 运行会自然重试。
3. **报告 URL 假设**：已核实 `src/web.py:50` 的 `/reports/{quarter}.html` 路由与设计一致。
4. **`QuarterComparator` 接口未核实**：已读取 `src/quarter_compare.py` 并确认接口与字段。
5. **消息长度无保护**：已增加 `_truncate_text` 截断策略和对应测试。
