## Architecture Decisions (locked in review)

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | 13F 用 **Adapter 包装**，不改 SEC13FFetcher | 已有代码已验证，零回归风险 |
| D2 | Claude **分步调用**：先并行摘要 → scoring → Top-N 交叉验证 | 成本可控，延迟可并行 |
| D3 | **部分容错**：单源失败标注降级，不阻塞整份报告 | 可用性 > 完美性 |
| D4 | **具体类优先**：SignalSource Protocol 仅作文档标注，不做强制抽象 | YAGNI——3 个实现不够抽象 |
| D5 | **异步并行 + SEC 串行**：13F+8-K 共享 EDGAR 限流，新闻 RSS 独立并行 | 10 req/s 限流是硬约束 |

## Data Flow

```
  SEC13FFetcher ──→ 13FAdapter ──┐
  NewsSource ─────────────────────┼──→ SignalAggregator ──→ [Claude 摘要] ──→ SignalScorer ──→ [Claude 交叉验证] ──→ HTML Report
  Sec8kSource ────────────────────┘         ↑ 单源失败不阻塞                    ↑ 规则引擎: 时间×主体×可信度
```

## Failure modes

| Failure | Test? | Error handling? | User sees? |
|---------|-------|-----------------|------------|
| RSS 源超时 | [GAP] | 降级标注 "新闻不可用" | "新闻信号暂不可用" in report |
| SEC EDGAR 429 | [GAP] | 退避重试 3 次，仍失败降级 | "SEC 事件暂不可用" in report |
| 8-K XML 解析失败 | [GAP] | 跳过该条 8-K，记录 WARNING | 信号面板少一条信号 |
| Claude API 超时 | [GAP] | 退避重试 3 次，仍失败降级 | "AI 分析暂不可用" in report |
| 全源空数据 | [GAP] | 正常产出报告 | "本季度无新增信号" |
| 新闻去重误判 | [GAP] | 无——重复信号可能展示两次 | 两条相似信号，非严重 |

## Parallelization Strategy

Sequential implementation — all new code touches the same `src/main.py` pipeline entry point. No independent lanes.

Recommend: single worktree, build in order: base.py → sources → aggregator → scorer → main.py integration → reporter.
