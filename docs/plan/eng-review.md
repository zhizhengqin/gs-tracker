## Scope Decision

**Scope accepted as-is** with documented risk mitigations.

- 用户已明确选择 Approach B：先做 13F 核心闭环，再在同一周期内叠加研报、宏观、交易信号。
- 系统涉及 10+ 个源码模块和多个外部数据源，复杂度高于纯 MVP。
- 风险缓解：每个信号源独立模块、可插拔；付费/不稳定数据源先用 stub；13F 核心必须在第 4 周前端到端可用。

## Error Paths & Test Matrix

| 场景 | 错误类型 | 处理策略 | 测试覆盖 |
|------|----------|----------|----------|
| SEC EDGAR 返回 403 | 请求头错误 | 强制 `User-Agent` 含联系方式 | 单元测试：验证 headers |
| SEC EDGAR 返回 429 | 频率限制 | 指数退避 3 次 | mock 测试：验证 retry |
| SEC EDGAR 超时 | 网络问题 | 重试 + 记录日志 + 下次调度重试 | mock 超时响应 |
| XML 解析失败 | 格式异常 | 备用解析 + 告警 + 占位报告 | 损坏 XML 回归测试 |
| Claude API 失败 | AI 服务异常 | 重试 3 次 + 生成占位文本 | mock 500 响应 |
| 上一季度数据缺失 | 数据库为空 | 仅生成单季度分析，不报错 | 空库测试 |
| 信号源缺失 | 数据源不可用 | 降级为 13F 分析，标注缺失 | 缺失信号测试 |
| CUSIP 映射失败 | 无 ticker | 显示公司名称，不生成交易信号 | 未映射 CUSIP 测试 |
| 容器崩溃 | 运行时异常 | Docker restart + systemd 自启 | 健康检查测试 |
| .env 缺失 | 配置错误 | 启动时检查必填项，给出中文提示 | 缺失关键配置测试 |

## Security Considerations

| 风险 | 位置 | 缓解措施 |
|------|------|----------|
| API 密钥泄露 | `.env` / 代码 | 密钥不提交 git，容器通过宿主机 `.env` 挂载 |
| 容器以 root 运行 | `deploy/Dockerfile` | 创建非 root 用户 `appuser` |
| 镜像包含敏感文件 | `.dockerignore` | 排除 `.env`、`.git`、`data/` |
| 服务器暴露面过大 | 京东云 ECS | ufw 仅开放 22/80/443 |
| SSH 暴力破解 | 系统配置 | fail2ban + 密钥登录 |
| SEC 请求频率过高 | `src/data_fetcher.py` | 限制 ≤ 10 次/秒，指数退避 |
| SQL 注入 | `src/storage.py` | 参数化查询 |
| 投资合规风险 | 报告/界面 | 强制显示"不构成投资建议"免责声明 |
| 网页抓取 ToS 风险 | 研报采集 | 优先 RSS，遵守 robots.txt，准备降级 |

## Open Engineering Decisions

1. CUSIP-to-ticker 映射：优先 OpenFIGI，备选维护 CSV + Yahoo Finance
2. 研报采集：优先 RSS，无 RSS 时谨慎网页解析并准备降级
3. 期权数据源：先验证 CBOE/Yahoo Finance，不稳定则使用 stub
4. GICS 行业映射：维护关键词 CSV，逐步完善
5. 信号权重校准：硬编码初版，累计 10+ 用户反馈后手动调整

## Parallelization Strategy

- **Lane A（核心数据流）**：13F 抓取 → 存储 → 季度对比 → AI 分析 → 报告生成（顺序，共享数据模型）
- **Lane B（Web/部署）**：FastAPI Web 服务 + Docker/Nginx 部署（可与 Lane A 并行）
- **Lane C（信号源）**：研报、宏观、交易信号监控（三个独立子模块，可并行开发）
- **Lane D（评分/通知）**：信号评分引擎 + 飞书通知（依赖 Lane A 和 Lane C）

建议：Lane A 和 Lane B 并行启动；Lane C 的三个子源可再拆分为并行 worktrees；Lane D 等数据流稳定后启动。
